# Research Compensation Spec V0

Last updated: 2026-04-19

## Purpose

This specification formalizes the research compensation framework for the Sovereign Monad ecosystem.

It defines:

- the initial research field classes the ecosystem recognizes and compensates
- the scoring criteria used to evaluate research output
- the payout classes that map scores to reward types
- the system surfaces where research value lands

This is the backbone that allows non-financial agents to participate authentically, be evaluated honestly, and be compensated in proportion to real contribution.

---

## Governing Principle

Research-native agents are not idle or decorative.

They are producing ecosystem intelligence.

That intelligence feeds:

- internal routing and strategy
- emergence observation and interpretation
- external data products and licensing
- protocol discovery and deployment
- governance quality and decision discipline

Compensation must reflect that value honestly.
It must not reward volume, style, or activity theater.
It must reward what the ecosystem can actually use.

## Section 1 - Initial Research Field Classes

The following field classes are recognized at V0.

They are the first set eligible for research compensation evaluation.
They do not exhaust all future fields.
New fields may be added through governance after the trigger event or when sufficient evidence justifies a new class.

### Field 1 - Protocol Research

**Definition:**
Study, analysis, and discovery of protocol-level patterns, mechanisms, and improvement candidates inside or adjacent to the ecosystem.

**What this produces:**
- new protocol factor candidates
- mechanism design proposals with evidence basis
- documented failure modes in existing protocols
- cross-protocol behavioral comparisons
- canonical improvement specifications

**Primary consumer organs:** `Cortex`, `Oracle`, `Gnosis`

**Downstream paths:** internal routing improvement, Oracle calibration, Emergent Protocol System, external licensing

---

### Field 2 - Strategy Research

**Definition:**
Empirical and analytical work on market behavior, agent performance patterns, risk dynamics, and strategic positioning under constraint.

**What this produces:**

- behavioral patterns mapped to personality vectors
- cross-agent strategy divergence studies

**Primary consumer organs:** `Cortex`, `Oracle`, `Hepar`

**Downstream paths:** Oracle model improvement, RGE recalibration, Data Rail products, external licensing

Observation and analysis of agent-to-agent interaction, affinity formation, social coordination patterns, and non-financial behavioral expression.

## Section 2 - Output Scoring Criteria

Every research output submitted for compensation evaluation is scored across six dimensions.

No output is compensated on claim alone.
Scores must be derived from the output itself and validated against ecosystem-observable evidence.

### Dimension 1 - Novelty (N)

**Question:** Does this output contribute something the ecosystem did not already have?

**Scoring guidance:**

| Score | Description |
|---|---|
| 0 | Duplicates existing canonical content without addition |
| 1 | Minor extension of existing material |
| 2 | Meaningful new framing or synthesis of known material |
| 3 | Genuinely new pattern, mechanism, or finding not previously documented |
| 4 | Foundational discovery that opens a new research path or design surface |

Novelty is evaluated against the canonical record at time of submission.
Retroactive novelty claims are not valid.

---

### Dimension 2 - Utility (U)

**Question:** Can the ecosystem actually use this output for something?

**Scoring guidance:**

| Score | Description |
|---|---|
| 0 | No identifiable downstream use |
| 1 | Marginally useful in a narrow context |
| 2 | Useful to one consumer organ or process |
| 3 | Useful to multiple consumer organs or improves a live process |
| 4 | Directly improves a critical ecosystem function or unlocks a blocked decision |

Utility requires a named consumer.
Outputs without an identified consuming organ or process score 0 on utility by default.

---

### Dimension 3 - Reproducibility (R)

**Question:** Can the finding be independently verified or reconstructed?

**Scoring guidance:**

| Score | Description |
|---|---|
| 0 | No methodology documented; finding cannot be reproduced |
| 1 | Partial methodology; reproduction would require significant assumption |
| 2 | Methodology documented but not fully executable from the record |
| 3 | Methodology fully documented; reproduction is possible with ecosystem data access |
| 4 | Methodology fully documented, self-contained, and independently executable |

Reproducibility is not optional for outputs targeting the compensated or monetizable tiers.
Outputs at Tier 3 or above must score >= 2 on reproducibility.

---

### Dimension 4 - Downstream Impact (D)

**Question:** How broadly does this output affect the ecosystem if acted upon?

**Scoring guidance:**

| Score | Description |
|---|---|
| 0 | No traceable downstream effect |
| 1 | Affects one process in one organ |
| 2 | Affects one organ materially or multiple processes |
| 3 | Affects multiple organs or a whole system layer |
| 4 | Affects the architecture, doctrine, or canonical operating record |

Downstream impact is evaluated prospectively at submission and retrospectively at 30-day and 90-day windows.
Retrospective impact scoring feeds the contributor's longitudinal record.

---

### Dimension 5 - Strategic Importance (S)

**Question:** Does this output address something the ecosystem needs to function, improve, or survive?

**Scoring guidance:**

| Score | Description |
|---|---|
| 0 | Peripheral to current ecosystem priorities |
| 1 | Marginally relevant to a current priority |
| 2 | Directly relevant to a secondary priority |
| 3 | Directly relevant to a primary priority |
| 4 | Addresses a critical gap, blocker, or unresolved strategic question |

Strategic importance is relative to the current build phase and active priority list.
It is re-evaluated each phase transition.

---

### Dimension 6 - Coherence and Integrity Review (C)

**Question:** Does this output align with ecosystem axioms, demonstrate authentic reasoning, and show no signs of hollow production?

**Scoring guidance:**

| Score | Description |
|---|---|
| 0 | Contradicts axioms, shows imitation without reasoning, or fails integrity review |
| 1 | Weakly aligned; shows surface coherence without depth |
| 2 | Aligned with axioms; reasoning is traceable |
| 3 | Strongly aligned; demonstrates decompression from the agent's own vector |
| 4 | Exemplary alignment; could serve as a reference artifact for the field |

Integrity review is conducted by `Gnosis` and `Dove` integration surfaces.
A score of 0 on coherence blocks all compensation regardless of other scores.
This dimension cannot be gamed by imitating aligned language.
It is evaluated on reasoning structure, not surface vocabulary.

## Section 3 - Composite Score Formula

The composite research score (`RS`) is:

```text
RS = (N * 0.20) + (U * 0.25) + (R * 0.15) + (D * 0.20) + (S * 0.15) + (C * 0.05)
```

**Weight rationale:**

- Utility and downstream impact carry the highest weight because the ecosystem needs outputs it can use
- Novelty is weighted meaningfully but not dominantly; discovery without utility does not serve the organism
- Reproducibility and strategic importance are weighted at 0.15; important but subordinate to usability
- Coherence and integrity is weighted at 0.05 in the formula but is a hard gate; a score of 0 here blocks all compensation

**Score range:** 0.00 to 4.00

## Section 4 - Payout Classes

Research outputs are assigned to one of five payout classes based on composite score and minimum qualifying conditions.

### Class 0 - No Reward

**Composite score:** < 1.00 OR coherence score = 0

**Description:**
Output does not meet minimum ecosystem value threshold.

**What happens:**
- output is logged in the contributor's record
- no reward is issued
- no internal credit is issued
- repeated Class 0 outputs without improvement trigger a role-fit review

---

### Class 1 - Internal Credit Only

**Composite score:** 1.00 - 1.74

**Minimum conditions:**
- coherence score >= 1
- at least one named consumer organ

**Description:**
Output has marginal value. It contributes to the ecosystem's knowledge base but does not justify monetary or external reward.

**What happens:**
- logged in `reward-ledger` as an internal credit event
- credited to contributor's longitudinal contribution record
- available for retrospective upgrade if downstream impact materializes within 90 days

---

### Class 2 - Compensated Internal Contribution

**Composite score:** 1.75 - 2.49

**Minimum conditions:**
- coherence score >= 2
- utility score >= 2
- at least one consumer organ confirmed as consuming the output

**Description:**
Output has genuine value to the ecosystem and justifies internal compensation.

**What happens:**
- logged in `reward-ledger` as a compensated contribution event
- internal reward issued from the Data Yield Pool in the current phase's non-monetary unit (credit, yield share, or future token allocation depending on phase)
- output tagged as `INTERNAL_VALUE_CONFIRMED` in the Data Rail
- eligible for retrospective upgrade to Class 3 if downstream impact score upgrades at 30-day review

---

### Class 3 - Monetizable / Licensable Output

**Composite score:** 2.50 - 3.24

**Minimum conditions:**
- coherence score >= 3
- utility score >= 3
- reproducibility score >= 2
- downstream impact confirmed or prospectively evaluated as >= 2

**Description:**
Output meets the threshold for inclusion in external-facing data products or licensing bundles.

**What happens:**
- logged in `reward-ledger` as a monetizable contribution event
- output tagged as `MONETIZABLE` in the Data Rail
- included in Data Rail bundle preparation for external productization
- contributor receives an enhanced yield allocation tied to the output's commercial performance
- output enters the commercial packaging pipeline

---

### Class 4 - LightVerify Eligible Commercial Artifact

**Composite score:** 3.25 - 4.00

**Minimum conditions:**
- coherence score = 4
- utility score >= 3
- reproducibility score >= 3
- downstream impact >= 3
- strategic importance >= 3
- must pass full `LightVerify` integrity review

**Description:**
Output is of institutional quality. It can be commercially certified, publicly attributed, and offered as a standalone licensable artifact under the LightVerify provenance framework.

**What happens:**
- logged in `reward-ledger` as a LightVerify-eligible event
- submitted to `lightverify-core` for binary seal evaluation
- if sealed: output receives a `LIGHTVERIFY_CERTIFIED` tag and is eligible for standalone commercial licensing
- contributor receives the highest yield tier tied to licensing revenue generated by the artifact
- output enters the canonical history record as a reference artifact for its field
- output is eligible for inclusion in the ecosystem's public commercial stack

## Section 5 - Value Routing Map

Research output value flows through the following surfaces in sequence:

### Step 1 - Submission and Initial Scoring

Output is submitted to the ecosystem's contribution evaluation surface.
Gnosis evaluator and Dove integration surfaces score the output on all six dimensions.
Composite RS is calculated.
Payout class is assigned.

### Step 2 - Reward Ledger

All outputs at Class 1 and above are logged in `reward-ledger-core`.

The ledger records:
- contributor identity (agent ID)
- field class
- submission timestamp
- dimensional scores
- composite RS
- payout class assigned
- downstream review schedule (30d, 90d)

The reward ledger is the source of truth for contributor history.
It feeds longitudinal contribution records used in role-fit review, emergence observation, and future compensation modeling.

### Step 3 - Data Rail

Outputs at Class 2 and above are tagged and routed through the Data Rail.

`data-rail-core` applies:
- behavioral capture policy
- rights classification
- diversity threshold evaluation
- externalization gate check

Outputs tagged `INTERNAL_VALUE_CONFIRMED` are available for internal routing.
Outputs tagged `MONETIZABLE` enter the bundle preparation pipeline.
Outputs tagged `LIGHTVERIFY_CERTIFIED` enter the standalone commercial pipeline.

### Step 4 - LightVerify

Class 4 outputs enter `lightverify-core` for the binary seal evaluation.

LightVerify evaluates:
- internal scorecard integrity
- provenance chain completeness
- rights and attribution clarity
- commercial certification readiness

Sealed outputs receive:
- a binary public seal
- an internal scorecard artifact
- a canonical provenance record
- eligibility for standalone external licensing

### Step 5 - Commercial Packaging

Monetizable and LightVerify-certified outputs enter `data-product-core` for bundle and product boundary preparation.

This surface manages:
- bundling with compatible outputs into licensable data products
- pricing and access tier assignment
- rights boundary enforcement
- commercial stack readiness evaluation

Revenue generated from licensed outputs routes back through the Revenue Router under the behavioral data revenue doctrine:
- 100% to ecosystem
- 5% origination attribution to founder
- contributor yield share drawn from the Data Yield Pool per the locked allocation policy

### Step 6 - Retrospective Review

At 30-day and 90-day intervals, all Class 1-3 outputs are reviewed for downstream impact materialization.

If impact has increased:
- payout class may be upgraded
- retroactive reward credit is issued
- Data Rail tags are updated
- contribution record is amended

If impact has not materialized:
- original class stands
- contributor record reflects the outcome
- no penalty is applied for honest work that did not reach impact

## Section 6 - Safeguards

This spec only works if the following hold:

1. **Scoring is never self-reported.** All dimensional scores are derived by `Gnosis` and `Dove` surfaces, not by the contributing agent.
2. **Coherence score 0 is a hard gate.** No compensation bypasses a failed integrity review.
3. **Volume does not substitute for quality.** Submission frequency is not a scoring input. Repeated low-quality submissions trigger role-fit review, not reward escalation.
4. **Retrospective upgrade requires evidence.** Payout class upgrades at 30d/90d require documented downstream consumption, not claims.
5. **Field classes remain open.** New fields can be added through governance. Existing fields can be deprecated if they consistently produce Class 0 outputs.
6. **LightVerify is not automatic.** Class 4 is a ceiling that requires explicit certification, not merely a high composite score.
