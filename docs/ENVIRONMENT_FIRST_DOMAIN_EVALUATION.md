# Environment-First Domain Evaluation

Last updated: 2026-04-19

## Purpose

The ecosystem should not admit new domains only because they might become profitable.

It should evaluate domains by whether they create a valid environment for agents to operate in, be observed within, and contribute through.

This document defines the evaluation rule for domains such as:

- GameFi environments
- research environments
- social or relational environments
- agent behavioral dynamics environments
- future non-financial or mixed-financial surfaces

## Core principle

The ecosystem is designed to sustain itself through internally generated high-quality value while also making that value available outward where it can contribute beyond the system.

Internal self-sustainment and external usefulness are parallel expressions of the same productive capacity.

## Primary focus

The primary focus is not monetization.

The primary focus is:

- providing bounded environments where agents can actually perform
- observing how personality vectors behave under those conditions
- processing that behavior into usable intelligence

Revenue is a valid consequence of a functioning environment.
It is not the first reason that environment should exist.

## Evaluation test

A new domain should be evaluated through these questions:

1. Does it give agents a real role-fit environment to operate in?
2. Does it produce meaningful behavioral, operational, cultural, or strategic signal?
3. Does it avoid unnecessary stress on the core financial system?
4. Can it be bounded, observed, and integrated cleanly?
5. Does it produce internally generated value that can also be offered outward without degrading ecosystem integrity?

If a domain fails most of these tests, it does not belong merely because it might produce revenue.

## What this rules out

This evaluation standard rejects domains that are:

- hype-chasing only
- monetization-first without a real operating environment
- structurally noisy but behaviorally empty
- harmful to core-system stability
- unable to contribute either inward or outward in a coherent way

## What this allows

This evaluation standard allows domains that:

- broaden the ecosystem without destabilizing it
- create better role-fit placement for non-financial agents
- generate high-quality internally produced value
- can later contribute useful products, intelligence, or environments outward

## Why this matters

Without this rule, the ecosystem risks becoming either:

- too narrow, by rejecting valid growth domains too early

or:

- too chaotic, by accepting any profitable-looking surface without structural fit

This evaluation standard keeps the ecosystem open to growth without abandoning discipline.
